Just Fucking Use Tmux

Your SSH dropped. Your laptop died. Your terminal crashed. Your work is gone. This didn't have to happen.

You're three hours into a database migration over SSH. Your WiFi hiccups for two seconds. Connection closed. The migration? Who fucking knows. Your carefully arranged terminal layout? Gone. The 500 lines of output you needed to review? Evaporated into the void.

This is what happens when you don't use tmux. And it's completely preventable.

What Tmux Actually Does

Tmux is a terminal multiplexer. It runs on the server (or your machine), not in your terminal emulator. Your terminal is just a window into tmux. Close the window? Tmux keeps running. SSH drops? Tmux keeps running. Laptop bursts into flames? Tmux. Keeps. Running.

Without Tmux

SSH connection drops → Process killed → 3 hours of work lost → You cry

With Tmux

SSH connection drops → Reconnect → tmux attach → Everything exactly where you left it

The Alternatives Are Garbage

GNU Screen
Released in 1987. Ctrl+a conflicts with readline. The last person who solved your Screen problem did it in 2009 on a mailing list that no longer exists.
Multiple Terminal Windows
Zero persistence. Close your laptop and everything dies. 47 windows across 6 workspaces and you have no idea which one has your dev server.
iTerm2 Splits
Mac-only. No detach/reattach. Doesn't work when you SSH to a server. Which is when you actually need it.
VS Code Terminal
Switch projects and your terminals die. SSH disconnects and your terminals die. Everything dies. Always.
Multiple SSH Connections
Resource waste. Connection limits. Re-authenticating constantly. No shared state. Amateur hour.

What You Get

  • Session persistence. Start a job, detach, go home, reattach. Your scrollback, your layout, your running processes — all there.
  • Multiplexing. Split panes. Multiple windows. Multiple sessions. One SSH connection, unlimited terminals.
  • Scriptable environments. One command to spin up your entire dev setup. Same layout every time.
  • 50,000 lines of scrollback. That error message from 20 minutes ago? Scroll up and find it. Search for it. Copy it.
  • Remote pairing. Two people attach to the same session. Both see the same thing. Both can type. No screen sharing latency.
  • Works everywhere. Linux, Mac, WSL, servers, containers. Same tool, same workflow, same muscle memory.

The Only Commands You Need

# Start a new session
tmux

# Start a named session
tmux new -s project-name

# Detach from session
Ctrl+b d

# List sessions
tmux ls

# Reattach to session
tmux attach

# Split pane vertically
Ctrl+b %

# Split pane horizontally
Ctrl+b "

# Navigate panes
Ctrl+b arrow-keys

That's it. That's the whole thing. You can learn the rest later. Or never. These commands alone will save your ass more times than you can count.

Stop Making Excuses

"The keybindings are weird."
They're fine. Ctrl+b, then a key. You'll have it memorized in a day. Remap them if you want, tmux doesn't care.

"I don't work on remote servers."
Cool, use it locally. Session persistence works the same way. Reboot your machine, your tmux sessions survive (with resurrect plugin). Your terminal windows don't.

"My terminal emulator has splits."
Can you detach from them and reattach from another machine? Can you SSH to a server and use them there? No? Then they're not the same thing.

"I'll learn it later."
You've been saying that for years. It takes 15 minutes to learn the basics. The ROI is infinite.
